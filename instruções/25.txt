# Technical Stack
Version: 1.0

---

# Objetivo

Definir as tecnologias oficiais utilizadas no desenvolvimento do Conecta Flow.

Nenhuma tecnologia deve ser adicionada sem avaliação.

O objetivo é reduzir complexidade e manter consistência.

---

# Princípios

A stack deve priorizar:

- velocidade de desenvolvimento
- baixa manutenção
- escalabilidade gradual
- comunidade madura
- integração com IA
- baixo custo operacional

---

# Aplicação Principal

## Framework

Next.js

Responsabilidades:

- frontend
- backend routes
- server components
- API layer

---

## Linguagem

TypeScript

Configuração:

strict mode obrigatório.

Nunca utilizar JavaScript.

---

## Interface

React

Responsável por:

- componentes
- estados visuais
- interações
- experiência do usuário

---

## Estilo

Tailwind CSS

Motivos:

- velocidade
- consistência
- integração com componentes
- facilidade com IA

---

## Component Library

shadcn/ui

Utilizar como base.

Nunca criar componentes básicos novamente sem necessidade.

---

# Formulários

Utilizar:

React Hook Form

+

Zod

Responsabilidades:

- validação
- schemas
- mensagens de erro

---

# Estado

Regra:

Priorizar estado local.

Utilizar soluções globais apenas quando necessário.

---

## Server State

Utilizar:

TanStack Query

Responsável por:

- cache
- sincronização
- loading states
- mutations

---

## Client State

Preferência:

React Context

Caso necessário:

Zustand

Nunca criar estados globais sem justificativa.

---

# Banco

## Plataforma

Supabase

Responsável por:

- PostgreSQL
- Auth
- Storage
- Realtime
- Edge Functions

---

# ORM / Acesso a Dados

A decisão oficial deverá ser definida antes da implementação.

Opções avaliadas:

Drizzle ORM

ou

Supabase Client

Critério:

simplicidade.

Evitar abstrações desnecessárias.

---

# Autenticação

Supabase Auth.

Responsável por:

- login
- sessão
- recuperação
- identidade

---

# Autorização

Nunca confiar apenas no frontend.

Utilizar:

- RLS
- permissões no domínio
- validações backend

---

# Storage

Supabase Storage.

Utilizado para:

- imagens
- anexos
- documentos
- arquivos

Nunca armazenar arquivos diretamente no banco.

---

# Realtime

Supabase Realtime.

Usos:

- novas mensagens
- atualização de status
- eventos da conversa

---

# Busca

Inicialmente:

PostgreSQL Full Text Search

Evolução:

serviço especializado caso necessário.

---

# Banco Vetorial

Inicialmente:

pgvector no Supabase.

Responsável por:

- embeddings
- conhecimento
- RAG

---

# IA

Arquitetura baseada em adaptadores.

Anthropic

OpenAI

Google

Outros futuros.
