# System Modules
Version: 1.0

---

# Objetivo

Este documento define os módulos oficiais do Conecta Flow.

Cada módulo possui responsabilidades próprias.

Nenhum módulo deve assumir responsabilidades pertencentes a outro.

Esta divisão servirá como base para:

- arquitetura;
- interface;
- banco de dados;
- APIs;
- workflows;
- permissões;
- desenvolvimento futuro.

---

# Filosofia

Existe apenas um produto.

O produto é dividido em módulos.

Alguns módulos pertencem ao Core.

Outros poderão ser adicionados futuramente.

Todos devem compartilhar a mesma arquitetura.

Nunca existirão versões diferentes do sistema.

---

# CORE

Os módulos abaixo fazem parte obrigatoriamente do Core.

Todo cliente utilizará estes módulos.

---

# Módulo

## Home

### Objetivo

Recepcionar o usuário.

Apresentar o estado atual da operação.

Não é um dashboard analítico.

É um painel operacional.

---

### Responsabilidades

- boas-vindas;
- notificações;
- atividades recentes;
- indicadores resumidos;
- conversas pendentes;
- follow-ups pendentes;
- próximos compromissos;
- economia gerada pela IA.

---

### Nunca faz

- análises profundas;
- relatórios completos;
- BI.

---

# Relação entre módulos

Todos os módulos se conectam através da Conversa.

Nunca diretamente entre si.

Exemplo.

Contato

↓

Conversa

↓

Agenda

↓

Pagamento

↓

Campanha

↓

Dashboard

A conversa sempre permanece como entidade central.

---

# Crescimento

Todo novo módulo deverá responder:

- Qual problema resolve?

- Qual módulo utiliza?

- Pode ser removido sem afetar o Core?

Caso a resposta seja negativa, provavelmente ele pertence ao Core e deverá ser reavaliado.
