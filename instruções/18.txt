# Module Architecture
Version: 1.0

---

# Objetivo

Definir a arquitetura de módulos do Conecta Flow.

A plataforma deve crescer através de módulos independentes.

O Core nunca deve depender de módulos opcionais.

Novos módulos devem poder ser adicionados sem alterar a arquitetura principal.

---

# Filosofia

O Core resolve o problema principal.

Os módulos expandem capacidades.

O cliente nunca compra um sistema diferente.

Ele apenas desbloqueia novas funcionalidades.

---

# Estrutura

Plataforma

│

├── Core

│

├── Module 1

├── Module 2

├── Module 3

├── Module N

Todos compartilham o mesmo domínio.

Nenhum módulo altera o Core.

---

# Company Modules

Cada empresa possui sua própria configuração.

Empresa A

Agenda

Empresa B

Agenda + Pagamentos

Empresa C

Core apenas

---

# Feature Flags

Todo módulo deve utilizar Feature Flags.

Nunca remover código para desativar funcionalidades.

---

# Permissões

Permissões pertencem ao módulo.

Exemplo.

appointments.create

appointments.edit

appointments.cancel

payments.generate

campaigns.execute

Nunca criar permissões genéricas.

---

# Banco

Cada módulo possui suas próprias tabelas.

Nunca misturar tabelas do Core.

---

# Eventos

Todo módulo consome eventos.

Todo módulo publica eventos.

Nunca acessar diretamente outro módulo.

---

# Dependências

Permitido.

Módulo

↓

Core

Nunca.

Core

↓

Módulo

---

# Dashboard

Os módulos podem adicionar indicadores.

Nunca modificar indicadores do Core.

---

# Navegação

A interface deve adaptar-se automaticamente.

Se o módulo não existir.

O menu desaparece.

Nenhuma tela quebrada.

---

# IA

O Conversation Agent poderá utilizar módulos como ferramentas.

Exemplo.

Consultar Agenda

↓

Criar Agendamento

↓

Publicar Evento

↓

Conversation continua

Nunca acoplar Agenda ao agente.

---

# Mission Control

Mission Control administra.

Licenças

Versões

Ativação

Saúde

Custos

Nunca implementa regras específicas do módulo.

---

# Evolução

Todo novo módulo responde.

Resolve um problema novo?

Ou apenas adiciona complexidade?

Caso não agregue valor claro.

Não deve existir.

---

# Regra Fundamental

O Core deve permanecer funcional mesmo que todos os módulos opcionais sejam removidos.

Os módulos expandem a plataforma.

Nunca redefinem sua essência.
