# Database Model
Version: 1.0

---

# Objetivo

Definir o modelo lógico do banco de dados do Conecta Flow.

Este documento especifica as entidades persistidas.

Não contém SQL.

Não contém migrations.

Não contém implementação.

As migrations deverão ser geradas a partir deste documento.

---

# Filosofia

O banco existe para servir ao domínio.

Nunca o contrário.

O modelo deve favorecer:

- simplicidade
- consistência
- isolamento
- auditoria
- escalabilidade

---

# Convenções

# Versionamento

Entidades críticas devem possuir campo version.

Facilita concorrência otimista.

---

# Auditoria

Nunca modificar audit_logs.

Nunca modificar timeline_events.

São registros imutáveis.

---

# Storage

Arquivos permanecem fora do banco.

Banco armazena apenas referências.

---

# Embeddings

Nunca armazenar embeddings junto das mensagens.

Utilizar coleção específica.

---

# Row Level Security

Obrigatório em todas as tabelas da empresa.

Mission Control utiliza políticas próprias.

---

# Backup

Backups devem preservar:

empresa

eventos

auditoria

integridade referencial

---

# Evolução

Novas tabelas devem responder.

Pertencem ao Core?

Pertencem a um módulo?

Pertencem ao Mission Control?

Nunca misturar responsabilidades.

---

# Regra Final

O banco deve refletir o domínio.

Nunca permitir que decisões de implementação alterem o modelo de negócio.
